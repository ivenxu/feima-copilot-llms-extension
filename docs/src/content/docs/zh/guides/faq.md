## 网络与连接

### Q: 需要梯子吗？

**A: 不需要。**

飞码扣使用国内服务器（阿里云），无需任何网络代理即可正常使用。所有模型请求都通过国内服务器路由，确保稳定快速。

---

### Q: 速度如何？

**A: 快速稳定。**

我们使用阿里云服务器，国内访问速度快，响应延迟低。具体响应时间取决于所选模型：
- **Qwen Flash**: 极速响应，通常 1-3 秒
- **Qwen3 Max**: 标准响应，通常 3-8 秒
- **思维链模型**: 深度推理需要更长时间，通常 5-15 秒

---

### Q: 支持哪些网络环境？

**A: 支持所有国内网络环境。**

- 企业网络
- 家庭宽带
- 移动网络
- 教育网

无需特殊配置，直接使用即可。

## VS Code 版本与兼容性

### Q: 支持哪些 VS Code 版本？

**A: 支持 VS Code 1.85.0 及以上版本。**

建议使用最新版本的 VS Code 以获得最佳体验。

---

### Q: 支持 VS Code Insiders 吗？

**A: 是的，完全支持。**

VS Code Insiders 版本与稳定版功能相同，飞码扣在两个版本中都可以正常使用。

---

### Q: 支持 VSCodium 吗？

**A: 目前不支持。**

飞码扣依赖 GitHub Copilot Chat 扩展，而 VSCodium 不支持该扩展。请使用官方 VS Code。

---

### Q: 支持 Web 版 VS Code 吗？

**A: 部分支持。**

在 vscode.dev 和 github.dev 上可以使用，但部分功能可能受限。建议使用桌面版 VS Code。

---

## 功能支持

### Q: 是否支持 Custom Agent？

**A: 是的，和原生 Copilot 一样支持。**

飞码扣完全支持 GitHub Copilot 的 Custom Agent 功能，使用方式与原生 Copilot 相同。您可以：
- 创建自定义 Agent
- 配置 Agent 行为和提示词
- 使用 Agent 执行特定任务

---

### Q: 是否支持 Skills？

**A: 是的，和原生 Copilot 一样支持。**

飞码扣完全支持 GitHub Copilot 的 Skills 功能，使用方式与原生 Copilot 相同。您可以：
- 使用内置 Skills
- 创建自定义 Skills
- 通过 Skills 执行特定操作

---

### Q: 是否支持 constitution.md、prompts.md 和 instructions.md？

**A: 是的，完全支持。**

飞码扣支持所有 GitHub Copilot 的自定义配置文件，包括：
- **constitution.md** - 定义 Agent 的行为准则和原则
- **prompts.md** - 自定义提示词模板
- **instructions.md** - 提供特定任务的详细指令

这些文件的使用方式与原生 Copilot 完全相同，放置在 `.github/copilot/` 目录下即可生效。

---

### Q: 是否支持工具调用（Tool Calls）？

**A: 是的，支持工具调用。**

所有聊天模型都支持工具调用功能，包括：
- 代码执行
- 文件操作
- API 调用

---

### Q: 是否支持思维链（Thinking）？

**A: 是的，多个模型支持思维链。**

支持思维链的模型：
- Qwen3 Max（32K 思维 tokens）
- Qwen3.5 Plus（80K 思维 tokens）
- DeepSeek V3.2
- GLM-5
- GLM-4.7
- MiniMax M2.5（32K 思维 tokens）
- Kimi K2.5（16K 思维 tokens）

思维链功能可提高复杂问题的解决质量。

---

### Q: 是否支持多语言？

**A: 是的，支持多语言。**

飞码扣支持：
- **中文**：完整中文界面和文档，模型对中文优化
- **英文**：完全支持，适合国际项目
- **其他语言**：模型支持多种编程语言和自然语言

---

### Q: 是否支持代码补全？

**A: 是的，支持代码补全。**

飞码扣提供：
- **聊天补全**：在 Copilot Chat 中获取代码建议
- **内联补全**：通过 Qwen Coder Turbo 模型（内测免费）

---

### Q: 是否支持代码审查？

**A: 是的，支持代码审查。**

推荐使用：
- **Qwen3 Max**：深度分析，32K 思维链
- **GLM-5**：高级推理，结构化输出

## 模型与计费

### Q: 如何计费？

**A: 按次计费，无订阅压力。**

计费公式：
```
加权请求次数 = 模型倍率 × 上下文倍率
```

- **模型倍率**：从 0.1x（Qwen Flash）到 2.0x（Qwen3 Coder Plus）
- **上下文倍率**：根据上下文大小从 0.5x 到 2.0x

详细计费信息请参阅[计费指南](./billing)。

---

### Q: 公测期间如何获取免费额度？

**A: 多种方式获取免费额度。**

- **每周赠送**：每周自动获得 50 点积分
- **邀请好友**：邀请朋友注册，双方获得奖励
- **参与推广**：定期活动赚取额外积分

[了解如何赚取更多额度 →](https://feimacode.cn/promotion)

---

### Q: 如何查看剩余额度？

**A: 三种方式查看。**

1. **状态栏**：启用 `feima.showQuotaInStatusBar` 设置
2. **命令面板**：`Ctrl+Shift+P` → "飞码: 查看账号"
3. **网站**：访问 [feimacode.cn/dashboard](https://feimacode.cn/profile)

---

### Q: 额度用完后怎么办？

**A: 多种解决方案。**

- **等待每周赠送**：每周自动获得 50 点积分
- **邀请好友**：获得奖励积分
- **购买额度**：访问 [feimacode.cn/pricing](https://feimacode.cn/pricing)

**A: 根据任务类型选择。**

详细选择指南请参阅[使用模型](./using-models)。

## 数据隐私

### Q: 我的代码会被存储吗？

**A: 不会。**

代码仅在请求时发送给 AI 模型进行处理，不会被存储或用于其他目的。详见我们的[隐私政策](https://feimacode.cn/privacy)。

---

### Q: 对话记录会保存吗？

**A: 保存在本地，不上传服务器。**

对话记录保存在您的本地设备，不会上传到服务器。您可以：
- 在 VS Code 中查看历史对话
- 手动删除对话记录
- 对话记录仅在本地存储

---

### Q: 数据如何传输？

**A: 加密传输。**

- 所有请求使用 HTTPS 加密
- 认证使用 OAuth2 + JWT
- 数据不经过第三方服务器

---

### Q: 是否符合数据合规要求？

**A: 是的。**

飞码扣符合：
- 中国数据安全法规
- 用户隐私保护要求
- 企业数据合规标准

## 登录与认证

### Q: 如何登录？

**A: OAuth2 一键登录。**

1. 按 `Ctrl+Shift+P`
2. 输入 "飞码: 登录"
3. 浏览器打开认证页面
4. 使用 GitHub 账号登录
5. 授权后自动返回 VS Code

---

### Q: 支持哪些登录方式？

**A: 目前支持 GitHub 登录。**

未来计划支持：
- 微信登录
- 邮箱登录

---

### Q: 登录令牌会过期吗？

**A: 会过期，但会自动刷新。**

- **访问令牌**：15 分钟有效期，自动刷新
- **刷新令牌**：7 天有效期，过期后需重新登录

---

### Q: 如何注销？

**A: 通过命令面板注销。**

1. 按 `Ctrl+Shift+P`
2. 输入 "飞码: 注销"
3. 当前会话将被注销

---

### Q: 登录失败怎么办？

**A: 检查以下几点。**

1. **网络连接**：确保网络正常
2. **浏览器**：检查默认浏览器设置
3. **回调超时**：回调在 5 分钟后过期，请快速完成登录
4. **防火墙**：检查防火墙是否阻止重定向

如仍无法登录，请[联系我们](#反馈与支持)。

---

## 故障排除

### Q: 模型选择器中找不到飞码模型？

**A: 检查登录状态。**

1. 确保已登录：`Ctrl+Shift+P` → "飞码: 查看账号"
2. 检查输出面板是否有错误消息
3. 重新加载 VS Code 窗口

---

### Q: 请求失败或超时？

**A: 多种可能原因。**

1. **额度不足**：检查剩余额度
2. **网络问题**：检查网络连接
3. **模型限制**：检查会员等级是否满足模型要求
4. **上下文过大**：尝试减少上下文大小

---

### Q: 响应质量不满意？

**A: 尝试以下方法。**

1. **切换模型**：不同模型有不同优势
2. **提供更多上下文**：详细描述需求
3. **使用思维链模型**：复杂问题使用思维链
4. **优化提示词**：使用清晰、具体的提示词

---
